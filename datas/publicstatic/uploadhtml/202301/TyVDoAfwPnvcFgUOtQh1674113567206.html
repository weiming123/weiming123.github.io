git 常用命令系列 —— git diff 命令常用用法。
  掌握好git，为我们的项目、代码保驾护航。 —— 论口号的重要性

git 比较文件在工作区与暂存区的区别；如果暂存区没有这个文件，就比较工作区和最近提交的区别：

    命令：git diff <path>
    举例：git diff README.en.md

git 比较工作区的文件和已提交的某个版本的文件的区别：

    命令：git diff [<commit>] <path>

    举例1：git diff 6271090266de046f75c336556e92601dc6dba8d4 README.md

    举例2：git diff 6271090266de046f75c336556e92601dc6dba8d4
    比较当前工作区的文件和某个版本的区别

git 两个版本里的文件的区别：

    命令：git diff [<commit1>] [<commit2>] <path>

    举例1：git diff d8fab0379a76534c63b61b0e5b446233121dbde3 2e219cfa47cf5bb379904f9793435ca59ba5e512 README.en.md

    举例2：git diff d8fab0379a76534c63b61b0e5b446233121dbde3 2e219cfa47cf5bb379904f9793435ca59ba5e512
    tips：比较两个版本之间的区别

git 比较两个文件之间的区别：

    命令：git diff --no-index <path1> <path2>

    举例1：git diff --no-index README.en.md README.md

    举例2：git diff README.md ../study/README.md

  tips：当在由Git控制的工作树中运行命令并且至少有一条路径指向工作树之外时，或者在由Git控制的工作树之外运行命令时，可以省略--no index选项。

git 比较暂存区和版本库的区别：

    命令：git diff [<options>] --cached [<commit>] [<path>…​]

    举例1：git diff --cached README.md
    tips：这里是比较暂存区的 README.md 文件和当前版本库最新提交的区别

    举例2：git diff --cached 6271090266de046f75c336556e92601dc6dba8d4 README.en.md
    tips：这里是比较暂存区的 README.en.md 文件和 62710902 版本库 README.en.md 文件的区别
